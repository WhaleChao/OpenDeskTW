# Word 文件中心

全能文件工作台 2.6.2 的 Word 文件中心以桌面版 Word 的工作分類為參考，但不複製其商標或介面。使用者先選「想完成什麼」，工作台先檢查與備份，再把文件交給適合的本機開源編輯引擎。

## 工作分類

| 分類 | 支援工作 |
| --- | --- |
| 檔案 | 新增、開啟、最近文件、儲存／另存、列印、文件資訊、版本備份、PDF |
| 常用 | 字型、文字效果、段落、定位點、樣式、清單、格式刷、貼上、尋找取代、分散對齊、智慧補齊、中文標題重編與 MAGI 摘要 |
| 插入 | 封面、分頁、表格、圖片、形狀、圖表、SmartArt、連結、書籤、交互參照、頁首頁尾、頁碼、文字物件、方程式 |
| 繪圖 | 畫筆、螢光筆、擦除、套索、墨跡轉換與簽閱註記 |
| 設計 | 主題、色彩、字型、樣式集、段落間距、浮水印、頁面色彩與框線 |
| 版面配置 | 邊界、方向、紙張、欄、分頁／分節、行號、斷字、縮排、物件排列、章節頁碼 |
| 參考資料 | 目錄、註腳／尾註、引文、書目、標號、圖表目錄、交互參照、索引與文件結構 |
| 郵件 | 信封、標籤、資料來源、合併欄位、條件規則、預覽與完成合併 |
| 校閱 | 拼字文法、字數、翻譯、註解、追蹤修訂、接受／拒絕、比較／合併、保護與無障礙 |
| 檢視 | 閱讀／整頁／草稿、文件地圖、尺規、格式標記、縮放、多頁、分割、並排與專注模式 |
| 進階 | 表單控制項、欄位、DOCM／VBA 保留、增益集界線、簽章、文件檢查、PDF 與 MAGI V2／V3 |

## 工作台原生能力

- 解析 DOCX／DOCM 的 OOXML，建立可點選閱讀的文件地圖與交付前報告。
- 辨識繁體中文四層標題編號，先建立時間戳記備份，再輸出重新編號的同格式副本。
- 重新封裝時原樣複製未修改的 ZIP 項目，因此 DOCM 的 VBA 專案與其他未處理物件不會被主動移除。
- 使用隔離的 LibreOffice 使用者設定檔轉換 PDF，避免干擾使用者正在執行的 LibreOffice 工作階段。
- MAGI 分析只連線當下唯一運作的 V2 或 V3；主視窗與 ONLYOFFICE「MAGI」頁籤都能顯示結果，後者可直接分析選取文字或整份目前文件。
- ONLYOFFICE 原有 AI 以台灣繁中相容副本保留為「其他 AI 模型」，不會被誤標成 MAGI。
- Word 相容快捷鍵直接在編輯器內執行；格式複製、行距、標題樣式與中文標題重編均有文件記憶體 LIVE 讀回測試。

## 相容性承諾

全能文件工作台的目標是完整處理日常與正式文件工作，並保留 Office 格式。VBA、ActiveX、COM、IRM、Microsoft 365 雲端共同編輯，以及部分複雜 SmartArt、3D 或專有欄位沒有開源等價實作；App 會保留原檔並明確警示，不會宣稱已完全相容或靜默轉換。

## 驗證標準

自動測試必須涵蓋四層標題、跨 XML 文字片段重編、目錄、頁碼、表格、註腳／尾註、註解、追蹤修訂、書籤與合併欄位。LIVE 測試另以真實 DOCX 完成「解析 → 重編 → 重新封裝 → 再解析 → PDF」全流程，並同時檢查 ONLYOFFICE、LibreOffice 與 MAGI。
